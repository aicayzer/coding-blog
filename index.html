Practice

Hello again! Today I want to discuss something a little different from the previous two posts (I touched on it a little in the last post). What I want to discuss is practice. Practice is key. Practice may seem obvious and practically every course on the internet you do will involve you writing your own code. This isn’t enough though because it just requires short term memory most of the time and that is useless in the long run.

Before I progress any further, I just want to address those of you readers who are considering skipping this post and suggest you don’t. I know you’ll think practicing is easy, and it is, but in reality not that many people actually bother from my experience.

First why practice? There is the obvious reason of it helps you move things from long term to short term memory but actually there are a lot of good reasons. One of these is that you’ll often find you have forgotten what you have just learnt. When I first started coding I was pretty confident that I had picked up everything I had learnt and after about a week or so an arrogant voice in my head was telling me I basically knew everything there was to know about HTML and CSS; oh how wrong I was. You see when I tried to code my first website independently of any course about two weeks in and I went to write “” I couldn’t. I was so used to Codecademy pre-writing it I literally didn’t even know how to tell the computer I was writing in HTML. THIS IS A VERY GOOD EXAMPLE OF WHY PRACTICE IS KEY!! So, in short, practice is good because often you assume you know things when you don’t really know them.

Reason 2: It makes you use StackOverflow. After realising in the above example that I had forgotten how to write the fist line of HTML, the first thing I did was go to StackOverflow and no surprises, the answer was there. This was pretty easy to find on StackOverflow but as I have mentioned before it is good to “practice” using stack as it is such a large data base and if you have never used it before and you try and find something relatively complicated of the bat it is surprisingly tricky. I have only addressed StackOverflow here but this is applicable to trying to find anything on google when it comes to coding.

Reason 3: You learn more as you practice. When I went to code my first site, I realised I didn’t know how to set a picture as the background (and nor should I have as it hadn’t come up in my course yet) but I still wanted to do it. So, a quick Google search and I found the code and to this day I remember how to set a picture as the background. This is just one example of many.

Reason 4: You develop references. Often when I am coding now and I can’t remember something I don’t go straight to Google, I go to websites which I have created in the past where I know I have written the term I am trying to think of. Obviously I could find the answer online but this is a much easier and quicker way to find what I am looking for often.

(Semi-)Reason 5: It get’s you excited about coding when you have just started. When you just start learning to code it is easy to lose interest as you learn lists and tables but the second you actually create a functioning website it gives you a whole new kick about coding.

So briefly, how to practice? Well I have sort of covered that in the above but… just set yourself a target be it to code a basic photo site or to write a python script for rock paper scissors and do it.

The Greatest.

Hello again! Today I want to address the next step I took. This step was from Codecademy to Udemy. Spoiler, Udemy is great. Really great. The greatest.

First, I want to address what Udemy is. Web Developer Bootcamp is a website where you can do courses. These courses range from photography to web developer bootcamps. The latter is what I was interested in. The courses are usually about £10/$11 (They are always 90% off, do not be fooled). Most of the web development ones are 40+ hours and it is essentially university style teaching. You watch, take notes, practice and then test yourself.

So, why do I think this is good? I think it is good for a few reasons. The first is very simple and a little personal. I liked having a teacher who would point out key information as opposed to having to read endless pages of Web Developer Bootcamp about it. I also found it easier to focus with a teacher as often I was learning late at night after school and my attention span was quite short by that point! The second reason is similar to the first reason. You can go at your own pace by changing the playback speed from 1x - 2x and you can easily skip out lectures which seem relativly basic/you already know about. This is really important as it means you can learn a lot in a short amount of time. Something which isn't true with Web Developer Bootcamp. Finally, once you have bought the course it is yours. Yours forever. Unlike other sites like Treehouse. This means you can always go back to a lecture if you are having a problem and I now personally use it as my go to (after StackOverflow) if I can't remember how to do something.

For those of you who this interests. I really recommend Colt Steel. I did the course he has on Web Developer Bootcamp and it was fantastic. I would first do the Web Developer Bootcamp and then either the advanced one after or move onto his other ones such as Python (I am going to come back to this in a future post). Click here to see the Web Developer Bootcamp to see that specific course. So, what can be learnt here? I REALLY recommend Web Developer Bootcamp and Colt's course in particular - it is the only basic web developer bootcamp I have done on there so I can't vouch for any others. However, I do want to mention something at this point. It is all very well learning but you must practice. This is what the next post will be on. Stay tuned for the next post.

Welcome back!!

Welcome to post two. Here I am going to address my jump into the programming world and how I first started, part 1.

At first, I started using Codecademy. This was great as it really wetted my appetite for coding and the beauty of what you can create with just a few lines out code.

The positives of Codecademy for me were:

  • You learn good syntax and formalities. What I mean by this is you learn a good way to structure your code which is so important if you ever want to collaborate or write long scripts.
  • The course content is really good. It makes sure you learn the essentials in the right order and someone has clearly spend a lot of time creating the syllabus
  • Codecademy is a company which makes money. It therefore has to compete with other “learn to code” sites so it must differentiate itself and it must be the best. In order to do this it keeps updating the syllabus as new iterations of languages come out. This is great as there is no point in learning outdated code.
  • Codecademy just has a good user interface (UI). There is no denying it. It is easy to navigate and easy to understand

The negatives are:

  • It is slow moving. There is no getting around that.
  • There will come a time when you want to branch out and begin to learn marginally more specialised things. This may not be the right way to put it. What I mean is that you may want to learn more about creating a toolbar than how to write a list. You can skip forwards but they often will try and make you practice the skills you have already learnt.
  • Finally, it doesn't teach you the most valuable coding skill in my opinion. Googling things. This may seem stupid as everyone is capable of using Google (or some other web browser). You will have had to use it to get here. But looking for more specific things on Google requires a little bit more competence. You need to be able to be very specific in a very short amount of words to find the references you are looking for regarding your problem - 95% of the time the answer will be somewhere on StackOverflow.

So do I recommend Codecademy. Yes and no. What I mean by this is, like said previously, it is great to wet your appetite for coding but not great past then as you ended up waisting time and you will become bored with something which is most certainly not a boring subject.

Welcome. Welcome to the blog.

Welcome. I learnt to code around a year ago. I learnt using any free content I could find on the internet; be it using blog posts or Codecademy. What struck me as I went along is there was a lack of documentation about what to learn/do and why. I found myself learning things because I just thought it would be useful to know. This was fine initially as learning HTML, CSS and Javascript is a pretty solid base. Beyond that, I struggled for a bit.

You see once you’ve learnt these languages and created a few websites you reach a huge decision. Do you proceed doing this or do you learn new languages, which ones, where and how do you test yourself. This last point was the most interesting for me. With the frontend trio (HTML, CSS and Javascript) you can just create websites which gradually increase in difficulty and size but with other languages with Python, it’s tricky to find challenges for a beginner. Sure if you are a pro there is enough things to do it would require multiple lifetimes but if you’re not, well…

So this “blog” aims to set out languages worth learning, why, how to learn and how to test yourself. I hope you enjoy…
